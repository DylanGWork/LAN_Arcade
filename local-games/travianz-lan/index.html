Should You Play?

Pick this when you want the fuller multiplayer Travian-style loop rather than Pillage First's solo offline world. It is heavier, older, and server-backed, but it is the closest match to the browser village strategy genre.

What it is

TravianZ is a browser strategy game inspired by classic Travian. Players register accounts, choose a tribe, build villages, train troops, trade, form alliances, and attack or raid other villages.

How it runs here

The LAN Arcade extracts the cached GPL source archive into a VM service folder, starts MariaDB and PHP Apache containers, and keeps the web game on local port 8092.

Current gate

This is a service needs game files, not a lightweight camping game. It should be started when players want it, smoked, and stopped when not in use if resources get tight.

Quick Manual

The first session is about getting a stable village before trying combat.

Register a player

Create a local account, choose Romans, Teutons, or Gauls, then log in from the LAN address.

Balance production

Upgrade wood, clay, iron, and crop fields so one resource does not bottleneck every build queue.

Unlock troops

Build the early military chain, train a small force, then use reports to learn whether raids are profitable.

Use the map

Nearby villages, oases, and NPCs are targets for scouting, diplomacy, trading, or attacks once the server rules allow it.

Admin notes

Start with scripts/prepare_travianz_service.sh start, stop with scripts/prepare_travianz_service.sh stop, and smoke with node qa/travianz-smoke.mjs. phpMyAdmin is only started with the script's admin-start action.
